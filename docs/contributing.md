# Contributing

## Branch Naming

Use feature branches when working on larger changes.

Examples:

* feature/authentication
* feature/workspaces
* feature/file-management
* feature/search
* feature/ai-assistant

## Commit Messages

Format:

```txt
<type>: <description>
```

Examples:

```txt
feat: add workspace creation
fix: resolve file upload issue
refactor: simplify auth middleware
docs: update architecture document
test: add workspace api tests
chore: setup docker environment
```

## Development Process

1. Create a branch
2. Implement the feature
3. Update relevant documentation if needed
4. Commit changes
5. Merge into main

## Documentation

Keep documentation simple and up to date.

Update documentation when:

* Architecture changes
* Major features are added
* Important decisions are made
